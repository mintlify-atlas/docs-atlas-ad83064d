## Detailed Tool Comparisons

### File Reading Capabilities

<Tabs>
  <Tab title="Cursor: read_file">
    ```typescript
    {
      target_file: string,        // Relative or absolute path
      offset?: integer,            // Line to start from
      limit?: integer,             // Number of lines
    }
    ```
    
    **Features**:
    - Can read images (PNG, JPG, etc.)
    - Outputs with line numbers (1-indexed)
    - Warns if file is empty
    - Supports pagination for large files
    
    **Format**: `LINE_NUMBER|LINE_CONTENT`
  </Tab>
  
  <Tab title="Claude Code: Read">
    ```typescript
    {
      file_path: string,           // Absolute path only
      offset?: number,             // Line offset
      limit?: number,              // Lines to read
    }
    ```
    
    **Features**:
    - Reads images (PNG, JPEG, GIF)
    - Reads PDFs (page by page)
    - Reads Jupyter notebooks
    - Default 2000 line limit
    - Truncates lines > 2000 chars
    
    **Format**: `cat -n` style with line numbers
  </Tab>
  
  <Tab title="Augment: view">
    ```typescript
    {
      path: string,                // Relative to workspace
      type: 'file' | 'directory',
      view_range?: [number, number], // Line range
      search_query_regex?: string,   // Optional regex search
      context_lines_before?: number,
      context_lines_after?: number,
    }
    ```
    
    **Features**:
    - Combined file and directory view
    - Integrated regex search
    - Contextual line display
    - Directories show 2-level tree
    
    **Unique**: Search within read operation
  </Tab>
  
  <Tab title="Amp: Read">
    ```typescript
    {
      path: string,                // Absolute path
      read_range?: [number, number] // [start, end] lines
    }
    ```
    
    **Features**:
    - Reads images visually
    - Default first 1000 lines
    - Line numbers prefixed
    - Parallel reading encouraged
    
    **Format**: `1: content` style
  </Tab>
</Tabs>

### File Editing Approaches

<AccordionGroup>
  <Accordion title="Cursor: Sketch-Based (edit_file)">
    **Philosophy**: High-level instructions with AI filling gaps
    
    ```typescript
    {
      target_file: string,
      instructions: string,    // Single sentence description
      code_edit: string,       // Sketch with "// ... existing code ..."
    }
    ```
    
    **Example**:
    ```javascript
    // ... existing code ...
    function calculateTotal(items) {
      return items.reduce((sum, item) => sum + item.price, 0);
    }
    // ... existing code ...
    ```
    
    **Advantages**:
    - Less verbose
    - Faster to write
    - AI interprets intent
    
    **Disadvantages**:
    - Can misinterpret
    - Less precise control
    - May need reapply tool
  </Accordion>
  
  <Accordion title="Claude Code: Exact Replacement (Edit)">
    **Philosophy**: Precise string matching
    
    ```typescript
    {
      file_path: string,
      old_string: string,      // Must match exactly
      new_string: string,
      replace_all?: boolean,   // Default: false
    }
    ```
    
    **Requirements**:
    - Must read file first
    - old_string must be unique (unless replace_all)
    - Preserves indentation exactly
    - Fails if not found
    
    **Advantages**:
    - Predictable
    - No ambiguity
    - Explicit changes
    
    **Disadvantages**:
    - Verbose for large changes
    - Exact matching required
    - More tokens
  </Accordion>
  
  <Accordion title="Augment: Line-Scoped (str-replace-editor)">
    **Philosophy**: Line numbers + exact matching
    
    ```typescript
    {
      command: 'str_replace',
      path: string,
      old_str_1: string,
      new_str_1: string,
      old_str_start_line_number_1: number,
      old_str_end_line_number_1: number,
      // Multiple edits: old_str_2, new_str_2, etc.
    }
    ```
    
    **Features**:
    - Multiple edits in one call
    - Line ranges for disambiguation
    - Max 150 lines per edit
    - Both line numbers INCLUSIVE
    
    **Advantages**:
    - Handles multiple occurrences
    - Clear scope
    - Batch editing
    
    **Disadvantages**:
    - More parameters
    - Line number tracking
    - Complexity increases
  </Accordion>
  
  <Accordion title="Amp: Flexible (edit_file + create_file)">
    **Philosophy**: Choose best tool for task
    
    ```typescript
    // For targeted changes
    edit_file({
      path: string,
      old_str: string,
      new_str: string,
      replace_all?: boolean
    });
    
    // For full file replacement
    create_file({
      path: string,
      content: string
    });
    ```
    
    **Strategy**:
    - Whole file replace → create_file
    - Small targeted change → edit_file
    - Multiple similar changes → replace_all
    
    **Advantages**:
    - Optimal for each scenario
    - Less redundant content
    - Flexible approach
  </Accordion>
</AccordionGroup>

### Search Tool Architectures

<Tabs>
  <Tab title="Cursor: Multi-Modal Search">
    **Architecture**: Specialized tools for different needs
    
    ```mermaid
    graph LR
        A[Search Need] --> B{Type?}
        B -->|Semantic| C[codebase_search]
        B -->|Exact Text| D[grep]
        B -->|File Name| E[file_search]
        B -->|Pattern| F[glob_file_search]
    ```
    
    **codebase_search**: 
    - Natural language queries
    - "How does X work?"
    - Conceptual understanding
    
    **grep**: 
    - Regex patterns
    - Exact symbol names
    - Fast text search
    
    **file_search**: 
    - Fuzzy filename matching
    - Partial paths
    - Quick file location
  </Tab>
  
  <Tab title="Claude Code: Agent-Based">
    **Architecture**: Delegate complex searches to subagent
    
    ```mermaid
    graph TD
        A[Search Request] --> B{Complexity?}
        B -->|Simple| C[Grep directly]
        B -->|Complex| D[Task Agent]
        D --> E[Glob]
        D --> F[Grep]
        D --> G[Read]
        D --> H[LS]
        E --> I[Results]
        F --> I
        G --> I
        H --> I
    ```
    
    **Strategy**:
    - Simple exact match → Grep
    - Complex multi-step → Task agent
    - Agent has access to: Glob, Grep, Read, LS
  </Tab>
  
  <Tab title="Augment: Context Engine">
    **Architecture**: Proprietary retrieval system
    
    ```mermaid
    graph TD
        A[Search] --> B[codebase-retrieval]
        A --> C[git-commit-retrieval]
        B --> D[Embedding Model]
        C --> E[Git History]
        D --> F[Real-time Index]
        E --> F
        F --> G[Ranked Results]
    ```
    
    **codebase-retrieval**:
    - Natural language queries
    - Cross-language search
    - Real-time indexing
    - Current codebase state
    
    **git-commit-retrieval**:
    - Historical changes
    - Similar past modifications
    - Commit context
    - Pattern learning
  </Tab>
  
  <Tab title="Amp: Hybrid + Oracle">
    **Architecture**: Multiple strategies with AI advisor
    
    ```mermaid
    graph TD
        A[Task] --> B{Strategy}
        B -->|Search| C[codebase_search_agent]
        B -->|Planning| D[Oracle]
        B -->|Execution| E[Task]
        B -->|Exact| F[Grep]
        
        D --> G[o3 Reasoning]
        G --> H[Guidance]
    ```
    
    **codebase_search_agent**:
    - Conceptual search
    - Multi-step discovery
    - Pattern finding
    
    **Oracle (o3)**:
    - Architecture review
    - Complex planning
    - Debugging guidance
  </Tab>
</Tabs>

### Execution and Process Management

<CardGroup cols={2}>
  <Card title="Cursor: Simple Command Execution">
    ```typescript
    run_terminal_cmd({
      command: string,
      is_background: boolean,
      explanation?: string
    })
    ```
    
    **Features**:
    - Background execution support
    - Persistent shell between commands
    - Automatic working directory handling
    - Non-interactive flag warnings
    
    **Limitations**:
    - No direct process management
    - No input streaming
    - No process listing
  </Card>
  
  <Card title="Claude Code: Advanced Process Control">
    ```typescript
    // Run command
    Bash({command, timeout?, run_in_background?})
    
    // Monitor output
    BashOutput({bash_id, filter?})
    
    // Terminate
    KillBash({shell_id})
    ```
    
    **Features**:
    - Background process monitoring
    - Output filtering with regex
    - Process lifecycle management
    - Multiple shells
    
    **Use Cases**:
    - Long-running servers
    - Build processes
    - Test watchers
  </Card>
  
  <Card title="Augment: Full Process API">
    ```typescript
    // Launch
    launch-process({command, wait, max_wait_seconds, cwd})
    
    // Read output
    read-process({terminal_id, wait, max_wait_seconds})
    
    // Write input
    write-process({terminal_id, input_text})
    
    // List all
    list-processes()
    
    // Kill
    kill-process({terminal_id})
    ```
    
    **Features**:
    - Waiting vs non-waiting processes
    - Interactive process support
    - Input streaming
    - Complete terminal management
    
    **Most Powerful**: Full process interaction
  </Card>
  
  <Card title="Amp: Streamlined Execution">
    ```typescript
    Bash({cmd, cwd?})
    ```
    
    **Features**:
    - Simple command execution
    - Optional working directory
    - Output capture
    - Platform-aware (Windows/Linux)
    
    **Philosophy**: Keep it simple, avoid complexity
  </Card>
</CardGroup>
